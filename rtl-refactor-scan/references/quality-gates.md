# RTL-Refactor-Scan 质量门控

定义 Skill 执行过程中的质量检查点，确保输出可靠、可验证。

## Gate 1：输入验证

**检查项**：
- [ ] 待分析文件存在且可读
- [ ] 文件是 Verilog/SystemVerilog（.v / .sv）
- [ ] 用户已确认扫描范围（模块/时钟/接口）

**不通过处理**：暂停执行，向用户说明缺失信息。

---

## Gate 2：扫描完整性

**检查项**：
- [ ] 所有 5 个维度已扫描（可综合性/可实现性/时序优化/仿真上板一致性/EDA LA 友好性）
- [ ] 每个发现点已记录：位置（行号）、分类（A/B/C）、描述、影响评估
- [ ] 已检查混合 blocking/non-blocking 赋值
- [ ] 已检查 variable part-select 和 unpacked array

**不通过处理**：补充扫描，不进入 Phase 3。

---

## Gate 3：风险分类准确性

**检查项**：
- [ ] A 类风险：确实会导致综合/时序/上板问题
- [ ] B 类风险：确实会影响质量但不阻塞功能
- [ ] C 类风险：确实不应该修改（已注明原因）
- [ ] 无"风险等级标记错误"（如将 C 类标记为 A 类）

**不通过处理**：重新评估风险等级。

---

## Gate 4：重构计划安全性

**检查项**：
- [ ] 计划已输出并等待用户确认（无计划不修改）
- [ ] 已明确说明：是否改变时序 latency / 接口 / ready-valid 语义 / reset 行为
- [ ] 已提供回归测试命令和 PASS 判据
- [ ] 已提供回退方案

**不通过处理**：不执行修改，补充计划。

---

## Gate 5：修改质量

**检查项**：
- [ ] 一次只做一个明确的优化主题
- [ ] 保持 diff 可读，未做无关格式化
- [ ] 未删除有用注释
- [ ] 新增注释说明了"为什么这样改"
- [ ] 遵守 Verilog-2001 风格
- [ ] 未改变模块外部接口
- [ ] 未改变协议语义

**不通过处理**：回退修改，重新实施。

---

## Gate 6：回归验证

**检查项**：
- [ ] 回归测试已执行
- [ ] 测试结果符合 PASS 判据
- [ ] 如有测试失败，已分析失败原因（非"屏蔽问题"）

**不通过处理**：暂停重构，先修复测试失败。

---

## Gate 7：输出完整性

**检查项**：
- [ ] 审计报告已写入文件
- [ ] 修改计划已写入文件（如执行了修改）
- [ ] 最终报告已写入文件（如执行了修改）
- [ ] 最终报告包含：修改摘要/风险消除/剩余风险/下一步建议

**不通过处理**：补充输出文件。

---

## 自动检查（可选）

如果项目有 CI 环境，可以增加以下自动检查：

1. **文件存在性检查**：审计报告/修改计划/最终报告是否存在
2. **模板完整性检查**：报告是否包含所有必需章节
3. **风险分类统计**：A/B/C 类数量是否合理（A 类不应过多）

---

## 质量评分（用于持续改进）

| 维度 | 评分标准 | 权重 |
|------|----------|------|
| 扫描覆盖率 | 5 个维度是否都扫描了 | 20% |
| 风险分类准确性 | A/B/C 分类是否合理 | 30% |
| 重构计划完整性 | 是否包含回归测试和回退方案 | 20% |
| 修改质量 | 是否遵守约束（接口不变等） | 20% |
| 输出完整性 | 是否所有报告都已输出 | 10% |

总分 0-100，建议 ≥80 才发布到团队使用。
