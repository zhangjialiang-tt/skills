# Output Risk Profile — ai-plat-bridge

| 风险 | 严重度 | 防御 |
|---|---|---|
| 宽泛请求触发全 vault 扫描 | 高 | 未确定卡片类型前返回澄清，不访问文件 |
| 路径穿越或白名单逃逸 | 严重 | 规范化路径并验证类型目录包含关系 |
| 用户未授权却主动同步 | 高 | 只有显式写入请求才进入写流程 |
| 自动联动扩大范围 | 高 | 使用固定单向矩阵；缺少项目关联时只写日志 |
| 覆盖已有内容或未提交修改 | 严重 | 写前读取和状态检查，只做局部修改 |
| 重复日志或重复结论 | 中 | 写前语义去重，写后检查 |
| 未验证判断写成事实 | 高 | 保留 `[待验证]`、`[推测]`、`[假设]` 状态 |
| 修改模板或治理文件 | 高 | 模板和治理文件只读 |
| 继续形成旧记忆双轨 | 高 | 拒绝旧记忆访问，引导到项目卡或知识卡 |
| 泄露敏感信息 | 严重 | 写前脱敏检查，禁止写入凭证和未脱敏日志 |
| 验证失败却报告成功 | 高 | 每个目标写后重读，全部成功后才能报告 |

## 发布门槛

- 四类卡片路由和动作白名单评测全部通过。
- 澄清和拒绝用例确认没有文件访问权限。
- 联动矩阵、去重、缺失项目关联和当日日志创建规则通过。
- Skill 元数据校验通过。
- 旧同步级别和旧记忆文件名无残留引用。
