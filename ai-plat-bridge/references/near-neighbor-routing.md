# 近邻路由（防误触发）

| 请求特征 | 是否触发 | 依据 |
|---|---|---|
| "记录到工作台" / "查工作台" | ✅ 触发 | 用户显式要求 |
| "看今天干了什么" | ✅ 触发 | 用户显式要求 |
| "查项目状态" | ✅ 触发 | 用户显式要求 |
| "更新项目文件" | ✅ 触发 | 用户显式要求 |
| "查一下 AGENTS.md" | ❌ 不触发 | 信息查询，无工程动作 |
| "帮我查一下 XX 文档" | ❌ 不触发 | 信息查询，无工程动作 |
| 完成代码修改 + 用户说"同步到工作台" | ✅ 触发 | 用户明确要求 + 工程变化 |
| 完成代码修改 + 用户未提同步 | ⚠️ 判断 | 根据"主动写入条件"判断 |
| 纯聊天 / 日常问候 | ❌ 不触发 | 无工程动作 |
| 未落地的方案讨论 | ❌ 不触发 | 无工程状态变化 |
| 当前工作目录在 vault 内 | ❌ 不触发 | 直接按 AGENTS.md 操作 |
| 简单浏览文件 | ❌ 不触发 | 无工程动作 |
| 仅格式或注释修改 | ❌ 不触发 | 无工程状态变化 |

## 边界情况处理

| 情况 | 处理 |
|---|---|
| 用户说"记录一下"但未指定项目 | 先问"记录到哪个项目？" |
| 用户说"同步进展"但无实质内容 | 说明"无可记录内容"并列出可写项 |
| 工程变化涉及多个项目 | 分别创建/更新对应项目文件 |
| 用户说"你决定写不写" | 有实质内容则写，无则说明 |
