# 同步级别决策树

根据信息价值和工程影响选择最轻量的同步流程。

## 决策树

```
工程变化发生
    │
    ├─ 用户明确要求记录/同步？
    │   ├─ 是 → 按用户要求级别
    │   └─ 否 → 继续判断
    │
    ├─ 是否产生可持久化变化？
    │   ├─ 否 → 不写入（仅当用户明确要求时记录）
    │   └─ 是 → 继续判断
    │
    ├─ 变化类型？
    │   ├─ 小参数调整 / 局部修复 / 单次验证通过
    │   │   └─→ 轻量同步
    │   │
    │   ├─ 项目状态/目标/阻塞点/关键结论变化
    │   │   └─→ 标准同步
    │   │
    │   ├─ 独立任务 / 里程碑 / 架构决策 / 长期约定
    │   │   └─→ 深度同步
    │   │
    │   └─ 跨 session 决策 / 完整排查过程
    │       └─→ 深度同步
```

## 各级别操作速查

| 级别 | 操作 | 不包括 |
|---|---|---|
| **轻量** | 追加今日日志 + 按需更新项目文件当前状态/下一步 | 任务卡片、MEMORY.md |
| **标准** | 读取项目文件 → 更新相关章节 + 追加今日日志 + 按需更新已有任务卡片 | 创建/归档任务卡片、MEMORY.md |
| **深度** | 更新项目文件（状态/结论/决策/里程碑）+ 创建/归档任务卡片 + 追加日志 + 每日记忆 + 判断是否更新 MEMORY.md | — |

## 升级条件

| 从 | 到 | 条件 |
|---|---|---|
| 轻量 | 标准 | 产生新证据、新目标、状态变化或阻塞点变化 |
| 标准 | 深度 | 需要创建或归档任务卡片，或涉及里程碑、架构选择、长期约定 |
| 轻量 | 深度 | 独立任务完成、架构决策、完整排查过程 |

## 写入前检查清单

- [ ] 已确认目标项目文件
- [ ] 已读取目标文件现有内容
- [ ] 已选择最轻量同步级别
- [ ] 已确认变化是实质性的（非临时尝试）
- [ ] 已确认信息可写入（非密钥/密码/Token）
